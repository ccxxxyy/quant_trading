## 一、支持三类市场


| 市场 | 典型品种 | 数据来源 | 适合谁 |
|------|---------|---------|--------|
| **A 股** | 600519 茅台、000001 平安 | AkShare | 做股票的人 |
| **美股** | AAPL 苹果、TSLA 特斯拉 | yfinance | 做美股的人 |
| **中国期货** | au 黄金、rb 螺纹钢、IF 股指 | AkShare + CTP 网关 | **做期货的人** |

**期货用户推荐路径：**

1. ：用演示数据回测（不需要开户、不需要真实行情）
2. ：AkShare 拉取期货历史 K 线（如 `AU2412.SHFE`）
3. ：注册 [SimNow 模拟盘](https://www.simnow.com.cn)（免费），用 CTP 网关练模拟交易
4. ：策略稳定后再考虑期货公司实盘（需自行开户）

> 注意：回测引擎对期货的**保证金、开平、今昨仓**做了简化模拟；CTP 实盘网关已实现框架，真实下单需 SimNow/期货公司账号。

## 二、流程（5 步）

### 第 0 步：确认环境

```powershell
cd D:\PythonProjects\quant_trading
uv sync --extra dev --extra web --extra data
uv run pytest tests/ -q
```

看到 `55 passed` 即环境 OK。

### 第 1 步：启动 Web 界面

```powershell
.\start_web.ps1
```

浏览器打开：**http://127.0.0.1:8888**

### 第 2 步：第一次回测

1. 左侧点 **「回测实验室」**
2. 策略选 **双均线**
3. **勾选「使用演示数据」**（不需要下载真实数据）
4. 初始资金改成你想要的数（如 `100000`）
5. 点 **「运行回测」**

你会看到：
- **权益曲线**：钱随时间怎么变
- **回撤曲线**：最坏时候亏多少
- **10 项指标**：收益率、夏普、最大回撤等
- **交易记录**：每一笔买卖

### 第 3 步：对比策略

回测页点 **「策略对比」**，系统同时跑 5 种策略，看谁历史表现最好。

### 第 4 步：命令行回测

新开一个终端（Web 那个别关）：

```powershell
cd D:\PythonProjects\quant_trading
uv run quant info
uv run quant backtest dual_ma --symbol DEMO.SSE --start 2024-01-01 --capital 100000
```

### 第 5 步：看策略说明

Web 左侧点 **「策略库」**，每张卡片是一种策略的说明和参数。

## 四、完整运行流程（从零到实战）

量化交易不是拿来就能赚钱的工具，而是一套**科学验证→反复测试→谨慎实战**的流程。下面是从零开始到确认策略可行的完整路径：

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        量化交易完整运行流程                              │
│                                                                         │
│  Phase 1: 准备数据                                                       │
│  ┌──────────┐                                                           │
│  │ ① 数据管理 │ → 下载历史K线（A股/美股/期货/场外基金）                     │
│  └────┬─────┘                                                           │
│       ↓                                                                 │
│  Phase 2: 策略验证（最核心，反复迭代）                                      │
│  ┌──────────┐   不满意     ┌──────────┐   不满意                          │
│  │ ② 回测实验 │ ──────→ 换策略 │ ③ 参数优化 │ ──────→ 调参数                │
│  └────┬─────┘   ↑         └────┬─────┘   ↑                              │
│       │满意     └──────────────┘│满意     └─────┘                        │
│       ↓                        ↓                                        │
│  ┌──────────┐                                                           │
│  │ ④ 蒙特卡洛 │ → 压力测试确认策略不是运气好                                │
│  └────┬─────┘                                                           │
│       │通过                                                              │
│       ↓                                                                 │
│  Phase 3: 模拟实战                                                       │
│  ┌──────────┐                                                           │
│  │ ⑤ 模拟盘   │ → 虚拟资金跑 1-3 个月，验证实际表现                         │
│  └────┬─────┘                                                           │
│       │确认可行                                                           │
│       ↓                                                                 │
│  Phase 4: 风控与保障                                                      │
│  ┌──────────┐   ┌──────────┐                                            │
│  │ ⑥ 风控中心 │ → │ ⑦ 监控告警 │ → 设好安全防线再考虑实盘                    │
│  └──────────┘   └──────────┘                                            │
└─────────────────────────────────────────────────────────────────────────┘
```

### Phase 1：准备数据

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 打开「数据管理」页 | 左侧导航栏点击 |
| 2 | 输入标的代码 | 如 `600519.SSE`（A股茅台）、`AAPL.NASDAQ`（美股苹果）、`161725.OTC`（招商中证白酒基金） |
| 3 | 选择日期范围 | 建议至少 2 年数据（如 2024-01-01 到今天） |
| 4 | 选择数据源 | A 股/基金选 AkShare，美股选 yFinance |
| 5 | 点击「获取并存储」 | 数据下载到本地，右侧「已存储标的」出现新条目 |
| 6 | 点击标的查看 K 线 | 确认数据完整，时间范围正确 |

> 如果不想下载数据，回测时可以勾选「使用演示数据」，用系统生成的模拟数据先体验。

### Phase 2：策略验证（最核心的阶段）

**第一步：初步回测**

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 打开「回测实验室」页 | |
| 2 | 选择策略 | 新手推荐「双均线」，有经验选 RSI/MACD/海龟 |
| 3 | 填入标的代码 | 与 Phase 1 下载的标的一致 |
| 4 | 设置初始资金 | 默认 30 万，可改成你计划投入的金额 |
| 5 | A 股勾选 T+1 + 前复权 | 让回测更贴近真实 A 股规则 |
| 6 | 点击「运行回测」 | 等待几秒，出现权益曲线和绩效指标 |

**怎么判断策略是否有效？**

| 指标 | 合格标准 | 不合格信号 |
|------|---------|-----------|
| 年化收益率 | > 8%（稳健）/ > 15%（积极） | 负数或接近 0 |
| 夏普比率 | > 1.0 | < 0.5 说明风险大于收益 |
| 最大回撤 | < 20%（保守）/ < 30%（激进） | > 40% 风险太高 |
| 胜率 | > 40% | < 30% 策略逻辑可能有问题 |
| 盈亏比 | > 1.5 | < 1.0 赚的比亏的少 |
| 交易次数 | > 30 次 | < 10 次样本不够不可信 |

**第二步：参数优化**

回测指标不够好？不急着换策略，先优化参数：

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 打开「参数优化」页 | |
| 2 | 选择同一个策略 | |
| 3 | 设置参数范围 | 如快线 5-30、慢线 20-60 |
| 4 | 点击「开始优化」 | 系统自动遍历所有参数组合 |
| 5 | 查看排名表 | 按夏普比率排序，找到最优参数 |
| 6 | 查看热力图 | 颜色越绿 Sharpe 越高，找绿色集中区域 |
| 7 | 用最优参数回到回测页重新验证 | |

**第三步：蒙特卡洛压力测试**

参数优化后指标好看了？但要确认不是运气好：

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | 回测完成后点击「蒙特卡洛压力测试」 | |
| 2 | 系统自动将交易序列随机重排 500 次 | |
| 3 | 查看分布图 | 观察 500 次模拟的最终资金分布 |
| 4 | 看 5% 分位数 | 即"最差情况下你还能剩多少钱" |
| 5 | 如果 5% 分位仍为正收益 | 策略较为稳健，可进入下一阶段 |
| 6 | 如果 5% 分位亏损严重 | 说明策略过度依赖特定交易顺序，需要重新调整 |

### Phase 3：模拟盘实战验证

回测和压力测试都通过后，进入模拟盘阶段。详见下方「模拟盘完整操作教程」。

建议至少运行 1-3 个月，观察：
- 策略在新行情下是否仍然有效
- 是否出现回测中没见过的极端情况
- 交易频率、滑点、手续费是否在预期范围内

### Phase 4：风控与安全保障

模拟盘验证通过后，配置好风控再考虑实盘：

| 步骤 | 页面 | 操作 |
|------|------|------|
| 1 | 风控中心 | 设置浮亏自动减仓阈值（如浮亏 5% 自动减半仓） |
| 2 | 风控中心 | 配置标的黑名单（排除不想交易的品种） |
| 3 | 风控中心 | 设置连续亏损暂停（如连亏 3 笔自动停策略） |
| 4 | 监控告警 | 配置告警推送（邮件/企业微信/钉钉） |
| 5 | 运维中心 | 设置定时任务（盘前数据更新、盘后报表生成） |
| 6 | 运维中心 | 启动进程守护（崩溃自动重启） |

> **重要**：当前系统主要用于回测和模拟，实盘需要对接券商（CTP/IB），详见 README「实盘对接状态」。

## 六、七种策略怎么选

| 策略 | 一句话 | 你如果是… |
|------|--------|----------|
| 双均线 | 趋势跟着走 | 新手首选，最简单 |
| 布林带 | 价格偏离就反向做 | 觉得价格在区间内震荡 |
| RSI | 涨太多卖、跌太多买 | 喜欢抓反弹 |
| MACD | 金叉买死叉卖 | 做中线趋势 |
| 海龟 | 突破新高追、跌破止损 | **期货趋势党**常用 |
| 网格 | 区间内低买高卖 | 期货/币圈震荡市 |
| 配对 | 两只相关品种价差套利 | 进阶，需要两只标的 |

> 没有「最好」的策略，只有「适不适合当前行情」。用**策略对比**功能横向比较。

## 七、回测结果指标怎么读

| 指标 | 通俗意思 | 参考 |
|------|---------|------|
| 总收益率 | 一共赚/亏了多少 % | 正数赚，负数亏 |
| 年化收益率 | 折算成一年的收益率 | >10% 算不错（演示数据仅供参考） |
| 夏普比率 | 每承担 1 份风险赚多少 | >1 可用，>2 很好 |
| 最大回撤 | 从最高点最多跌了多少 | 越小越好，<-20% 要警惕 |
| 胜率 | 赢的交易占比 | 50% 左右正常 |
| 盈亏比 | 总盈利÷总亏损 | >1 说明赚的比亏的多 |
| 交易次数 | 一共买卖了多少次 | 太少可能样本不够 |

### P0-1 ✅ 复权模式回测（A 股必选）

1. 左侧点 **「回测实验室」**

> **什么时候用？** 回测 A 股时，不勾选则默认允许 T+0（不符合 A 股规则）。

- 回测引擎内置 ±10% 涨跌停过滤（默认启用）

- 无需手动操作

1. 左侧点 **「风控中心」**

from quant_trading.core.guardian import ProcessGuardian

guardian = ProcessGuardian()
guardian.add_process(
    name="web-server",
    command=["uv", "run", "quant-web"],
    max_restarts=5,        # 最多重启 5 次

### P1-13 ✅ 事前四重风控规则

系统在每笔订单提交前自动执行四重检查，不合规的订单直接拒绝：

| 检查项 | 规则 | 默认阈值 |
|--------|------|---------|
| 单标的持仓比例 | 单标的持仓市值不超过总权益的 X% | 25% |
| 单笔下单比例 | 单笔金额不超过总资金的 X% | 10% |
| 下单频率 | 每分钟最大下单笔数 | 100 笔/分钟 |
| 日亏损比例 | 当日亏损达到此比例时冻结 | 5% |

**Web 操作步骤：**
1. 打开 **风控中心** 页面，滚动到 **「事前四重风控规则」** 面板
2. 查看 4 条规则的当前值和说明
3. 在 **修改** 列直接输入新阈值 → 失焦后自动保存
4. 底部显示当前冻结/策略暂停状态

> 这些规则**自动生效**，被拒订单可在 **「监控告警」** 页面看到 `order_rejected` 类型告警。

### P2-1 ✅ K 线叠加买卖点

回测完成后，系统自动在权益曲线上用三角形标注策略的买入（绿色 ▲）和卖出（红色 ▼）信号位置，直观展示策略何时入场、何时出场。

**Web 操作步骤：**
1. 打开 **回测实验室** → 配置策略参数 → 点击 **「运行回测」**
2. 滚动到 **「K 线叠加买卖点」** 面板
3. 图表自动展示权益曲线 + 买卖标记
4. 鼠标悬停可查看具体日期和权益值

> **什么时候看？** 每次回测后，检查策略是否在合理的位置买入卖出（比如是否在下跌时追高买入）。

### P2-2 ✅ Monte Carlo 压力测试

将回测产生的交易序列随机打乱 500 次重排，模拟"如果这些交易以不同的顺序发生"会怎样，生成最终资金和最大回撤的概率分布。

**Web 操作步骤：**
1. 打开 **回测实验室** → 先运行一次回测
2. 滚动到 **「Monte Carlo 压力测试」** 面板
3. 点击 **「运行模拟」** 按钮
4. 查看结果：
   - **KPI 卡片**：原始最终资金、5%/50%/95% 分位的最终资金、平均/最差最大回撤
   - **权益分布带图**：三条线分别代表最差(5%)、中位(50%)、最优(95%)路径
   - **最终资金分布柱状图**：每 5% 分位的最终资金值

> **什么时候用？** 当回测收益不错但担心运气成分时，Monte Carlo 能告诉你"最差情况下还能赚吗"。如果 5% 分位仍然盈利，策略相当稳健。

### P2-3 ✅ 复盘报表

自动生成按月汇总的盈亏统计、连胜/连亏分析、单笔交易分析等复盘数据。

**Web 操作步骤：**
1. 打开 **回测实验室** → 先运行一次回测
2. 滚动到 **「复盘报表」** 面板
3. 点击 **「生成报表」** 按钮
4. 查看结果：
   - **KPI 卡片**：最大连胜/连亏次数、最大单笔盈利/亏损、平均持仓天数、平均盈亏
   - **月度盈亏柱状图**：绿色=盈利月、红色=亏损月
   - **月度明细表**：每月的盈亏金额、交易次数、胜率

> **什么时候用？** 找出策略的"好月份"和"坏月份"，判断策略是否在特定时期失效。

### P2-4 ✅ 参数优化热力图

在参数优化页面运行优化后，回测实验室页面自动生成两个核心参数与 Sharpe 比率的颜色矩阵热力图。

**Web 操作步骤：**
1. 打开 **参数优化** 页面 → 配置参数搜索范围 → 点击 **「开始优化」**
2. 回到 **回测实验室** 页面，滚动到 **「参数优化热力图」** 面板
3. 表格以颜色深浅展示不同参数组合的 Sharpe 值
   - 绿色 = 高 Sharpe（好）
   - 红色 = 低 Sharpe（差）
4. 快速定位最优参数区域

> **什么时候用？** 参数优化后，看参数是否在"高原"上（周围参数也不错=稳健）还是在"尖峰"上（稍微改参数就变差=过拟合）。

### P2-5 ✅ 标的黑名单

将高风险或不想交易的标的加入黑名单，系统在事前风控检查时自动拒绝对这些标的的下单。

**Web 操作步骤：**

4. 点击标的右侧的 **「移除」** 按钮可解除黑名单

> **什么时候用？** 当某只股票出现重大负面消息、被 ST、或你在该标的上连续亏损不想再碰时，直接加黑名单。

### P2-6 ✅ 流动性过滤

设置最低成交量和最低成交额阈值，自动过滤流动性不足的品种，避免因无法成交或价格异常导致的风险。

**Web 操作步骤：**
1. 打开 **风控中心** → 找到 **「流动性过滤」** 面板
2. 设置 **最低成交量**（默认 10,000 股）和 **最低成交额**（默认 100 万元）
3. 勾选 **「启用流动性过滤」** 复选框
4. 点击 **「保存设置」**

> **什么时候用？** 交易小盘股或新上市股票时，成交量太低可能导致买入后卖不出去。建议始终开启。

### P2-7 ✅ 风控日报 & 连续亏损暂停

自动汇总当日盈亏、持仓、订单记录，并监控连续亏损笔数，超过设定阈值时自动暂停策略。

**Web 操作步骤：**
1. 打开 **风控中心** → 滚动到 **「风控日报 & 连续亏损暂停」** 面板
2. 设置 **连续亏损暂停阈值**（默认 3 笔），点击 **「保存阈值」**
3. 点击 **「生成日报」** 查看当日风控报告：
   - 账户余额、当日盈亏、下单数、持仓标的数
   - 当前连续亏损笔数、历史最大连亏、暂停阈值
   - 如果连续亏损达到阈值，会显示红色警告
4. 点击 **「手动检测」** 可立即触发连续亏损检测

> **什么时候用？** 每天交易结束后生成日报复盘；连续亏损暂停可防止情绪化交易导致亏损加剧。

### P2-8 ✅ 可视化配置管理

在 Web 界面直接编辑系统配置（系统/数据/风控/回测），保存后自动持久化到 `config/settings.yaml` 并实时生效。

**Web 操作步骤：**
1. 打开 **系统设置** 页面
2. 每个配置项都有输入框，直接修改值即可
3. 点击右上角 **「保存全部修改」** → 只有真正修改过的项才会写入文件
4. 如果配置搞乱了，点击 **「恢复默认」** 一键重置

配置分四组：
- **系统配置**：系统名称、日志级别、时区、数据目录
- **数据配置**：存储引擎、Parquet 目录、DuckDB 路径
- **风控配置**：持仓上限、下单上限、日亏损上限、频率上限
- **回测配置**：默认佣金率、滑点率、初始资金

> **什么时候用？** 调整风控参数、更换数据存储路径、修改回测默认资金等操作，不用手动编辑 YAML 文件。

### P2-9 ✅ 因子缓存管理

将计算好的因子数据持久化到 Parquet 文件，避免重复计算。

**Web 操作步骤：**

4. 缓存列表展示各标的的因子行数、因子数、文件大小
5. 点击每行的 **「删除」** 可清除单个缓存，或点击 **「清除全部」** 清空所有缓存

> **什么时候用？** 训练 ML 模型前，先缓存因子数据可加快后续加载；或需要长期保存因子结果时。

### P2-10 ✅ 数据目录 & SQL 查询

浏览本地所有已存储数据集的元数据，并可直接用 SQL 语句查询 K 线数据。

**Web 操作步骤：**

3. 在右侧 **「SQL 查询」** 面板输入 SQL 语句，例如：

4. 点击 **「执行查询」**，结果以表格形式展示

> **什么时候用？** 快速探索数据内容、统计行数、检查数据质量。视图名称格式为 `标的代码_周期`（如 `600519_1d`）。

### P2-11 ✅ 多账户管理

### P2-12 ✅ 配对交易分析工具

扫描多个标的之间的相关性和协整关系，推荐适合配对交易的组合。

**Web 操作步骤：**
1. 打开 **策略库** → 滚动到 **「配对交易分析工具」** 面板
2. 在输入框填写多个标的代码（逗号分隔），例如：`600519.SSE,000858.SSE,000568.SSE,600809.SSE`
3. 点击 **「扫描配对」**
4. 左侧展示 **相关性矩阵**（绿色=高正相关，黄色=中等，红色=负相关）
5. 右侧展示 **推荐配对表**：
   - 相关系数、协整检验 p-value、最优对冲比
   - 建议列：「可配对」（相关>0.7 且协整显著）/「弱」/「不建议」

> **什么时候用？** 想做配对交易但不知道选哪两只股票配对时。协整 p-value < 0.1 说明两者长期有均值回归关系。
