## 一、支持三类市场


| 市场 | 典型品种 | 数据来源 | 适合谁 |
|------|---------|---------|--------|
| **A 股** | 600519 茅台、000001 平安 | AkShare | 做股票的人 |
| **美股** | AAPL 苹果、TSLA 特斯拉 | yfinance | 做美股的人 |
| **中国期货** | au 黄金、rb 螺纹钢、IF 股指 | AkShare + CTP 网关 | **做期货的人** |

**期货用户推荐路径：**

1. **第 1 周**：用演示数据回测（不需要开户、不需要真实行情）
2. **第 2 周**：AkShare 拉取期货历史 K 线（如 `AU2412.SHFE`）
3. **第 3 周**：注册 [SimNow 模拟盘](https://www.simnow.com.cn)（免费），用 CTP 网关练模拟交易
4. **第 4 周及以后**：策略稳定后再考虑期货公司实盘（需自行开户）

> 注意：回测引擎对期货的**保证金、开平、今昨仓**做了简化模拟；CTP 实盘网关已实现框架，真实下单需 SimNow/期货公司账号。

## 二、流程（5 步）

### 第 0 步：确认环境

```powershell
cd D:\PythonProjects\quant_trading
uv sync --extra dev --extra web --extra data
uv run pytest tests/ -q
```

看到 `55 passed` 即环境 OK。

### 第 1 步：启动 Web 界面

```powershell
.\start_web.ps1
```

浏览器打开：**http://127.0.0.1:8888**

### 第 2 步：第一次回测

1. 左侧点 **「回测实验室」**
2. 策略选 **双均线**
3. **勾选「使用演示数据」**（不需要下载真实数据）
4. 初始资金改成你想要的数（如 `100000`）
5. 点 **「运行回测」**

你会看到：
- **权益曲线**：钱随时间怎么变
- **回撤曲线**：最坏时候亏多少
- **10 项指标**：收益率、夏普、最大回撤等
- **交易记录**：每一笔买卖

### 第 3 步：对比策略

回测页点 **「策略对比」**，系统同时跑 5 种策略，看谁历史表现最好。

### 第 4 步：命令行回测

新开一个终端（Web 那个别关）：

```powershell
cd D:\PythonProjects\quant_trading
uv run quant info
uv run quant backtest dual_ma --symbol DEMO.SSE --start 2024-01-01 --capital 100000
```

### 第 5 步：看策略说明

Web 左侧点 **「策略库」**，每张卡片是一种策略的说明和参数。

## 五、七种策略怎么选

| 策略 | 一句话 | 你如果是… |
|------|--------|----------|
| 双均线 | 趋势跟着走 | 新手首选，最简单 |
| 布林带 | 价格偏离就反向做 | 觉得价格在区间内震荡 |
| RSI | 涨太多卖、跌太多买 | 喜欢抓反弹 |
| MACD | 金叉买死叉卖 | 做中线趋势 |
| 海龟 | 突破新高追、跌破止损 | **期货趋势党**常用 |
| 网格 | 区间内低买高卖 | 期货/币圈震荡市 |
| 配对 | 两只相关品种价差套利 | 进阶，需要两只标的 |

> 没有「最好」的策略，只有「适不适合当前行情」。用**策略对比**功能横向比较。

## 六、回测结果指标怎么读

| 指标 | 通俗意思 | 参考 |
|------|---------|------|
| 总收益率 | 一共赚/亏了多少 % | 正数赚，负数亏 |
| 年化收益率 | 折算成一年的收益率 | >10% 算不错（演示数据仅供参考） |
| 夏普比率 | 每承担 1 份风险赚多少 | >1 可用，>2 很好 |
| 最大回撤 | 从最高点最多跌了多少 | 越小越好，<-20% 要警惕 |
| 胜率 | 赢的交易占比 | 50% 左右正常 |
| 盈亏比 | 总盈利÷总亏损 | >1 说明赚的比亏的多 |
| 交易次数 | 一共买卖了多少次 | 太少可能样本不够 |

```
刚入门 ──→ Web 演示回测 + 策略库

    │
    ├── 出异常要保命 ──→ Web「风控中心」→ 一键冻结 / 一键清仓 / 浮亏减仓
    │
    ├── 要分配仓位 ──→ Web「风控中心」→ 仓位管理器（等权/风险平价/凯利）
    │
    ├── 想跑实时策略 ──→ Web「实时策略」→ 启动 → 连接 WebSocket 行情源（或手动推 K 线）→ 查看状态
    │
    ├── 想看 AI 预测 ──→ Web「AI 实验室」→ 计算特征 → 查看因子
    │
    ├── 想监控风险 ──→ Web「监控告警」页 → 查看阈值和告警记录
    │
    ├── 做期货 ──→ SimNow 模拟盘（CTP 网关）
    │
    ├── 做美股 ──→ IB 网关 + TWS
    │
    ├── 想定时跑任务 ──→ TaskScheduler（盘前数据更新 / 盘后统计 / 间隔检查）
    │
    ├── 要保证不崩 ──→ ProcessGuardian（子进程监控 + 崩溃自动重启）

    └── 想写自己的策略 ──→ 继承 BarSeriesStrategy → 放 strategies/ 目录
```

## 九、全功能操作教程（P0 + P1）

> 以下每项功能均可在 Web 界面（http://127.0.0.1:8888）中完成，无需终端命令。

### P0-1 ✅ 复权模式回测（A 股必选）

1. 左侧点 **「回测实验室」**
2. 在"复权模式"下拉框选择 **前复权**（推荐）或 后复权
3. 点击 **「运行回测」**
4. 权益曲线使用的就是复权后的价格，消除了除权除息导致的价格跳变

> **什么时候用？** 回测 A 股必须选前复权，否则历史价格有跳变导致策略信号失真。

### P0-2 ✅ T+1 交易规则

1. 回测实验室页面，勾选 **「T+1」** 复选框
2. 运行回测 → 当日买入的股票，引擎会阻止当日卖出
3. 交易记录中可以看到：买入日和卖出日至少间隔 1 天

> **什么时候用？** 回测 A 股时，不勾选则默认允许 T+0（不符合 A 股规则）。

### P0-3 ✅ 涨跌停过滤

- 回测引擎内置 ±10% 涨跌停过滤（默认启用）
- 当 K 线收盘价触及涨跌停时，撮合引擎会拒绝成交
- 无需手动操作，自动生效

### P0-4 ✅ 印花税

- 回测引擎 A 股卖出时自动收取万五印花税（默认启用）
- 回测结果的「手续费」指标已包含印花税
- 无需手动操作

### P0-5 ✅ 紧急风控（冻结/解冻/暂停/恢复/一键清仓）

1. 左侧点 **「风控中心」**
2. 顶部 4 个 KPI 显示：冻结状态 / 策略状态 / 当日盈亏 / 下单笔数
3. 操作按钮：
   - **「冻结账户」** → 立即禁止所有下单（保命用）
   - **「解除冻结」** → 恢复下单
   - **「暂停策略」** → 停止策略信号触发，但不影响已有持仓
   - **「恢复策略」** → 恢复策略运行
   - **「一键清仓」** → 弹出确认框 → 确认后平掉所有持仓并冻结账户
4. 底部风控详情面板显示完整状态（frozen / strategies_halted / daily_pnl 等）

> **什么时候用？** 实盘运行中发现异常（大幅亏损、行情异动、系统故障），立即冻结+清仓保命。

### P0-6 ✅ 模拟盘交易

1. 左侧点 **「模拟盘」**
2. 顶部 KPI：总资产 / 可用资金 / 手续费 / 持仓数
3. 下单面板：填写标的、选择方向（买/卖）、类型（市价/限价）、数量、价格 → 点击 **「下单」**
4. 下方表格显示当前持仓和挂单
5. 点击 **「重置模拟盘」** 可自定义初始资金和费率重新开始

> **什么时候用？** 回测满意后，在真实行情环境下用虚拟资金验证策略。

### P0-7 ✅ 实时策略运行

1. 左侧点 **「实时策略」**
2. 选择策略（如"双均线"）、输入标的代码 → 点击 **「启动运行」**
3. 顶部 KPI 变为绿色"运行中"，显示策略名和标的
4. 在右侧"手动推送 K 线"面板输入价格和成交量 → 点击 **「推送 K 线」**
5. 推送记录实时滚动，"已接收 K 线"计数递增
6. 点击 **「停止运行」** 结束

### P1-1 ✅ 过户费

1. 左侧点 **「回测实验室」**
2. 在回测表单中找到 **「过户费率」** 输入框（默认 0.00002 = 万分之 0.2）
3. 可修改费率 → 运行回测
4. 回测结果的手续费中已包含过户费（买卖双向收取）

### P1-2 ✅ 浮亏自动减仓

1. 左侧点 **「风控中心」**
2. 找到 **「浮亏自动减仓」** 区域
3. 设置触发阈值（如 -0.05 表示浮亏 5%）和减仓比例（如 0.5 表示减半）
4. 点击 **「检测浮亏」** → 系统扫描所有持仓
5. 若有持仓浮亏超过阈值，返回减仓建议单（标的、方向、数量）
6. 若无需减仓，提示"无需减仓"

### P1-3 ✅ 仓位管理器（等权/风险平价/凯利/固定金额）

1. 左侧点 **「风控中心」**
2. 找到 **「仓位管理器」** 区域
3. 选择模式：
   - **等权**：每个标的分配相同金额
   - **风险平价**：按波动率反比分配（波动小的多买）
   - **凯利公式**：根据胜率和盈亏比计算最优仓位
   - **固定金额**：每个标的固定投入金额
4. 填入总资金和标的列表（逗号分隔） → 点击 **「计算目标仓位」**
5. 表格显示每个标的的权重、目标金额、目标股数（已按 100 股整手调整）

### P1-4 ✅ 追踪止损 & 条件单

- 这两种订单类型在**回测引擎**中自动支持：
  - **追踪止损（Trailing Stop）**：止损价跟随最优价格浮动，如价格创新高后回落 3% 则止损
  - **条件单（Conditional）**：满足自定义条件表达式（如 `close > 100`）后自动触发
- 在编写策略代码时使用 `OrderType.TRAILING_STOP` 或 `OrderType.CONDITIONAL` 下单
- 回测结果中可在交易记录里看到这些订单的执行情况

### P1-5 ✅ 告警推送（邮件 / 企业微信 / 钉钉 / 飞书）

通过代码配置告警通道（后续版本将提供 Web 配置界面）：

```python
from quant_trading.monitoring.alert import EmailAlertHandler, WebhookAlertHandler

# 邮件推送
email = EmailAlertHandler(
    smtp_host="smtp.qq.com", smtp_port=465, use_ssl=True,
    username="bot@qq.com", password="app_password",
    sender="bot@qq.com", recipients=["you@example.com"],
)
email.send("告警标题", "告警内容详情")

# 企业微信推送
wecom = WebhookAlertHandler(
    url="https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=YOUR_KEY",
    platform="wecom",
)
wecom.send("系统异常", "策略触发浮亏阈值，已自动减仓")
```

### P1-6 ✅ WebSocket 实时行情接入

1. 左侧点 **「实时策略」**，先启动一个策略
2. 在启动面板下方找到 **「WebSocket 行情源」** 区域
3. **URL**：填入实际行情源地址（默认 `ws://127.0.0.1:9999/ws/market`）
4. **标的列表**：留空则自动使用策略绑定的标的，也可手动填写（逗号分隔）
5. 点击 **「连接行情源」** → 提示"已连接"
6. 连接状态 KPI 变为绿色 **"已连接"**，重连次数为 0
7. 如果行情源断线，系统自动按 1s→2s→4s→...→60s 指数退避重连
8. 如果某标的超过 2 分钟无数据，系统触发缺失检测告警
9. 点击 **「断开行情源」** 手动断开

### P1-7 ✅ 数据中心：指数 / 可转债 / 期权 / Tick

1. 左侧点 **「数据管理」**
2. 标的代码输入框下方有品种示例提示：
   - 股票：`600519.SSE`
   - ETF：`510300.SSE`
   - 指数：`000300.SSE`（沪深 300）
   - 可转债：`113050.SSE`
   - 期货：`au2412.SHFE`
3. 周期下拉新增 **「Tick 逐笔」** 选项
4. 数据源下拉显示 **AkShare (A股/期货/指数/可转债)**
5. 填写标的和日期 → 点击 **「获取并存储」**
6. 系统自动识别标的类型并调用对应的拉取方法

### P1-8 ✅ 定时任务调度

1. 左侧点 **「运维中心」**
2. 顶部 KPI：调度器状态 / 定时任务数 / 守护器状态 / 被守护进程数
3. 左侧面板 **「定时任务调度器」**：
   - 点击 **「启动调度器」** → 调度器开始按时执行已注册的任务

4. 通过代码注册自定义任务：

```python
from quant_trading.core.scheduler import TaskScheduler
from datetime import time

scheduler = TaskScheduler()
scheduler.add_pre_market("更新行情", update_data_func, run_time=time(9, 15))
```

### P1-9 ✅ 进程守护 / 崩溃自动重启

1. 左侧点 **「运维中心」**
2. 右侧面板 **「进程守护器」**：
   - 点击 **「启动守护器」** → 开始监控所有已注册的子进程
   - 进程列表表格显示：进程名、状态（运行中/已崩溃/重启中）、PID、运行时间、重启次数
   - 每行有 **「重启」** 按钮，可手动重启指定进程
   - 点击 **「停止守护器」** → 终止所有子进程并停止监控
3. 通过代码注册被守护进程：

```python
from quant_trading.core.guardian import ProcessGuardian

guardian = ProcessGuardian()
guardian.add_process(
    name="web-server",
    command=["uv", "run", "quant-web"],
    max_restarts=5,        # 最多重启 5 次
    health_url="http://127.0.0.1:8888/api/health",  # 健康检查 URL
)
guardian.add_process(
    name="strategy-runner",
    command=["uv", "run", "python", "-m", "quant_trading.strategy.runner"],
    max_restarts=10,
)
```

> **崩溃重启机制：** 子进程非正常退出（exit code ≠ 0）时，按指数退避冷却重启（2s→4s→8s→...→120s），达到最大重启次数后停止并告警。

### P1-11 ✅ Walk-Forward 滚动验证（防过拟合）

Walk-Forward 用于防止策略参数在历史数据上"过度拟合"，确保策略在未来行情也能盈利。

**Web 操作步骤：**
1. 打开 **回测实验室** 页面，先配置策略、标的、日期（或使用默认）
2. 滚动到 **「Walk-Forward 防过拟合分析」** 面板
3. 设置 **训练窗口**（如 180 天）和 **测试窗口**（如 60 天）
4. 点击 **「运行 Walk-Forward」**
5. 查看结果：窗口数、平均测试 Sharpe、一致性比率（>50% 较稳健），以及各窗口明细表

**代码方式（可选）：**

```python
from quant_trading.alpha.walkforward import WalkForwardValidator

validator = WalkForwardValidator(
    strategy_id="dual_ma",
    symbol="600519.SSE",
    train_days=180,
    test_days=60,
)
result = validator.run(start=..., end=..., use_demo_data=True)
print(result.summary())
```

> **什么时候用？** 参数优化后、上实盘前，用 Walk-Forward 验证确认策略不是"死记历史"。

### P1-12 ✅ TWAP / VWAP 执行算法（大单拆分）

大额订单如果一次性下单，会冲击市场价格。TWAP/VWAP 把大单拆成小单分批执行。

**Web 操作步骤：**
1. 打开 **模拟盘** 页面，滚动到 **「执行算法拆单（TWAP / VWAP）」** 面板
2. 选择算法（TWAP 或 VWAP）、标的、方向、总数量
3. 设置 **切片数** 和 **间隔（秒）**（TWAP 用）
4. 点击 **「预览拆单方案」**
5. 表格展示每个切片的数量和执行时间/权重

**代码方式（可选）：**

```python
from quant_trading.execution.algorithms.twap import TWAPAlgorithm
from quant_trading.model.instrument import InstrumentId
from quant_trading.model.order import OrderSide

algo = TWAPAlgorithm(
    instrument_id=InstrumentId.from_str("600519.SSE"),
    side=OrderSide.BUY,
    total_quantity=10000,
    duration_minutes=30,
    num_slices=10,
)
```

> **什么时候用？** 单笔下单量 > 日均成交量 1% 时建议使用，避免市场冲击。

### P1-13 ✅ 事前四重风控规则

系统在每笔订单提交前自动执行四重检查，不合规的订单直接拒绝：

| 检查项 | 规则 | 默认阈值 |
|--------|------|---------|
| 单标的持仓比例 | 单标的持仓市值不超过总权益的 X% | 25% |
| 单笔下单比例 | 单笔金额不超过总资金的 X% | 10% |
| 下单频率 | 每分钟最大下单笔数 | 100 笔/分钟 |
| 日亏损比例 | 当日亏损达到此比例时冻结 | 5% |

**Web 操作步骤：**
1. 打开 **风控中心** 页面，滚动到 **「事前四重风控规则」** 面板
2. 查看 4 条规则的当前值和说明
3. 在 **修改** 列直接输入新阈值 → 失焦后自动保存
4. 底部显示当前冻结/策略暂停状态

> 这些规则**自动生效**，被拒订单可在 **「监控告警」** 页面看到 `order_rejected` 类型告警。
